Bird Maze

c++ visual studio code project

The aim of the project is to create a game using the tutorials given which lean towards Pacman. Bird maze is a spin on this where a bird has to collect seeds while being chased by cats. Bananas give a speed boost and the player can distract cats with a ball of wool for a few seconds.

This was made in Visual Studio and with SDL/S2D and wrritten in C++. The main focus was to code graphics and display them on the screen and have them animated. This project also demonstrates using arrays to display multiple objects, for example the seeds are displayed at random positions but are the same object loaded over and over and stored in an array. This project also demonstrates collisions and sound effects at a basic level. All the graphics were drawn myself.

Cheese Chase

C# Unity prototype platformer

This project was inspired by Cheese Land in Mario Kart and the sterotype of mice loving cheese. The idea was to have platforms made out of different types of cheese which caused different effects. There are swiss cheese platforms which collapse under the player, bouncy cheese jump pads and moudly cheese tiles which damage the player.

What is notable about this, is this is the first time I felt like I had a proper understanding of Unity. I learnt how to import my own sprite sheets and how to split them into drawable tiles on different layers. I also learnt how to have animations run due to states. The thing I will use in a lot of other projects is the UI, I made which is interactive as the heart sprites change depending on the player health and the text which changes to display score. This I can use for a health bar in any other game I make and would work for enemies as well. I also learnt about how to use IEnemurators to have real time waited before executing a command. This is how the platform falls down, as well as removing the rigid body.

Pizzeria Armagedon

C# Unity prototype top down shooter

Premise: You play as a chef in a pizzeria that is being attacked by zombie pizza dough. You must shoot the pizza dough with the correct pizza toppings before cooking it into a pizza to earn points and survive as long as possible.

I created a pizza themed tile set and have sliced it with a 72 by 72 grid. I have turned it into a drawable tile palette and created a restaurant.
It was a challenge to make the projectiles fire in the direction the player was facing. First I had to work out how the Vector 2 for player direction works. I created a debug to print out the coordinates of the player direction and converted it into a simpler int which is number 1-4 depending on direction. I then have the bullet communicate with the character controller and set another vector 2 to Vector2.right or Vector2.left etc to get the bullet to be in the same direction as the player. I then use transform.translate and this Vector2 with the stored direction to fire it in the same direction as the player. To fire the bullet I have both the new and the old input systems in the project and use the "jump" key to use space as fire. On pressing space the bullet is instantiated from a prefab with the direction set up on start.
The gun is a child of the character and uses a script to change the sprite to face the correct direction using the int of numbers 1-4. There are also 4 firepoints which are at the end of the gun and depending on the direction a different fire point is used to make the bullet come from the end of the gun.
I have implemented UI to swap between the different projectiles - This uses 3 buttons which when clicked, change a int to a number between 1 and 3 which in the shooting script changes which projectile is fired.
The enemies can collide with bullets which causes a child object to display a graphic for the sauce and cheese. Once hit by the flame, the movement script is deleted and the child objects display no graphics and the enemy turns into a pizza - the player can interact with this object to score points
The enemies have a spawning system which if there are no enemies on screen, at least one enemy must be spawned. Then there is a timer and enemies spawn every so often but only a certain amount can spawn at a time - the more points you score, the more can spawn at a time.

Marshall Eats Muffin Simulator

OpenGL 3D Game Engine in C++

This was created using the OpenGL libraries in visual studio and written in C++
WASD to move character and camera forward, backwards, left and right (figure 2) QE to move the character and camera up and down. The game is called Marshall eats muffins simulator as the playable character is a seal named Marshall who must eat muffins to score points. Colliding with muffins adds ten points whereas touching the pufferfish minuses 5 points. There is also a muffin count displayed on screen (figure 3) which gets updated. The seal and camera move together using WASD and QE to create a followable character. The muffins are rotating slowly by themselves and can be collided if the radius of the seal and muffin collide. This collision is the same with the pufferfish, however, these move on their own around the screen. The pufferfish movement path is randomised by randomising how long before they change direction and randomising the direction. The whole scene is in a sky sphere which is textured on the inside.
